📞 VoxLog Help

Quick Start

  1. Get a free Groq API key at console.groq.com → sign up → API Keys. Starts with gsk_.
  2. Enter your key in the modal that appears on load, or click 🔑 Groq Key in the header anytime.
  3. Load your WAV files — click 📁 Load WAV Files or drag-and-drop .wav files directly onto the drop zone.
  4. Transcribe — click ⚡ Transcribe All to process every file. Or click the individual ⚡ button on any row. Each file is sent to Groq Whisper and results appear in seconds.
  5. Review & edit — caller name, phone number, call date, and notes are all auto-filled and editable.
  6. Save your work — click 💾 Save .db to download a voxlog.db file you can reload later.
Tip: Demo mode (skip API key) lets you explore the full dashboard with sample data — useful for testing before you have a key.

Sorting & Search

Use the search bar to filter rows across all columns — phone, caller name, transcript text, notes, date, or filename.

Use the sort dropdown to order by phone number (default), date, caller name, filename, or status.

Status Counts (Header)

Total — all loaded voicemails. Done — rows marked Done. Pending — rows still Pending or In Progress.

Tip: After transcribing, quickly scan the Caller and Phone columns — the AI fills these in automatically but you can correct anything it missed. Changes are saved next time you click 💾 Save .db.

Saving Your Session

  1. After transcribing and editing, click 💾 Save .db in the toolbar.
  2. A file called voxlog.db will download to your computer.
  3. Keep this file somewhere safe — it contains all your voicemail records, transcriptions, caller info, notes, and statuses.

Loading a Saved Session

  1. Open VoxLog (this HTML file) in your browser.
  2. Click 📂 Load .db in the toolbar.
  3. Select your voxlog.db file — all your records restore instantly.
  4. You can then load additional WAV files on top and transcribe new ones without losing existing records.

What's Stored in the .db

FieldStored?
Phone, Caller, Date, Status✓ Yes
Full transcription text✓ Yes
Notes✓ Yes
Original .wav audio file✗ No — only metadata & text
Note: The original .wav audio files are not stored in the .db — only the transcription text and metadata. If you need to re-transcribe a file, you'll need the original .wav again.
Power tip: The .db file is a standard SQLite database. You can open it in DB Browser for SQLite or query it with any SQLite tool for custom reporting.

Export to CSV

  1. Click ⬇ Export CSV in the toolbar.
  2. A file called voxlog_export.csv downloads immediately.
  3. Open it in Excel, Google Sheets, or any spreadsheet app.
